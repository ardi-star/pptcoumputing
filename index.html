Cloud Computing

Memahami Struktur, Mekanisme, dan Keamanan Teknologi Awan

Apa itu Cloud Computing? (NIST Definition)

Sebelum masuk ke arsitektur, kita perlu memahami definisinya secara standar:

  • Model Layanan: Memungkinkan akses jaringan yang nyaman dan sesuai permintaan ke kumpulan sumber daya komputasi.
  • On-demand self-service.
  • Broad network access.
  • Resource pooling.
  • Rapid elasticity.
  • Measured service.

Hierarki Arsitektur Cloud

  • Front-End (Sisi Klien): Antarmuka dan aplikasi yang digunakan user.
  • Back-End (Sisi Provider): Infrastruktur fisik dan virtual.
  • Middleware: Komunikasi antara Front-End dan Back-End.

Komponen Front-End

  • Client Infrastructure: PC, laptop, mobile.
  • User Interface: Browser atau aplikasi.
  • HTTP / HTTPS untuk komunikasi jaringan.

Komponen Back-End

  • Application: Software yang merespon permintaan.
  • Service: SaaS, PaaS, IaaS.

Runtime Cloud & Middleware

  • Virtual Machines / Containers
  • Middleware untuk konektivitas dan database.

Virtualisasi & Hypervisor

  • Virtualization server
  • Hypervisor membagi server menjadi VM.

Cloud Storage & Database

  • Scalable Storage
  • Database SQL & NoSQL

Networking & Security

  • Virtual Private Cloud
  • Load Balancer
  • Firewall & IAM

Model Layanan

  • SaaS: Gmail
  • PaaS: Google App Engine
  • IaaS: AWS EC2

Deployment Models

  • Public Cloud
  • Private Cloud
  • Hybrid Cloud
  • Multi Cloud

Kesimpulan & Masa Depan Cloud

  • Virtualisasi dan otomatisasi meningkatkan efisiensi.
  • Serverless Computing
  • Edge Computing
  • AI Driven Cloud
